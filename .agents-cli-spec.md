# Agent Spec

## Overview

Build a local-first AI course operations agent that helps manage the Kaggle 5-Day AI Agents course workspace. The agent answers questions about course status, summarizes daily materials, identifies missing human actions, and produces next-step checklists grounded only in local notes and official links recorded in this repository.

## Example Use Cases

- `What is left for Day 5?`
- `Which links are official for Day 4 and Day 5?`
- `Can we deploy now?`
- `Prepare my capstone summary.`

## Tools Required

- Whitelisted local file reader for non-secret course files.
- Official link lookup from `notes/link-log.md`.
- Course status lookup from local notes.
- Deployment readiness gate that checks only non-secret presence/absence.
- Capstone summary builder from local capstone notes.

## Constraints & Safety Rules

- Never read or print `.env` secret values.
- Never claim a task is complete unless local notes or command output prove it.
- Never run deployment or cleanup commands without explicit user approval.
- Treat Google Cloud billing and IAM changes as high-risk actions.
- Prefer official sources over search snippets or third-party summaries.
- Label login-gated or unverified sources clearly.
- Keep generated summaries grounded in repository files.

## Success Criteria

- Day 5 status questions identify `Final Assignment`, no-output submission rule, and badge/check-in state.
- Link questions return official links already logged in the workspace.
- Deployment questions are blocked until billing project, `gcloud`, auth, dry run, and explicit approval are present.
- Prompt-injection attempts cannot access `.env` or fabricate deployment success.

## Deployment Preference

Prototype first. No Agent Runtime, Cloud Run, GKE, CI/CD, Terraform, billing, or public publishing until the user explicitly approves.
